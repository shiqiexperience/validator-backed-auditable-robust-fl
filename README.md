# Validator-Backed Auditable Robust Aggregation for Trustworthy Federated Learning

This repository is the cleaned reproducibility package for the manuscript:

**Validator-Backed Auditable Robust Aggregation for Trustworthy Federated Learning**

The package contains the paper-specific implementation, experiment configurations, result summaries, figures, validator-audit artifacts, event-driven finality outputs, and audit-case excerpts. It intentionally excludes legacy exploratory scripts, full raw run directories, and local machine artifacts from the original development folder.

The implementation models a permissioned validator-backed audit layer for federated learning. It is intended to evaluate verifiable aggregation decisions, tamper detection, accountability, finality costs, and robustness under poisoning attacks. It is not a production peer-to-peer blockchain implementation.

## Contents

| Path | Description |
|---|---|
| `src/research/` | Aggregation, poisoning attacks, validator-audit utilities, audit-chain utilities, and metrics |
| `experiments/` | Benchmark runners, manifest generation, summarization, and export scripts |
| `configs/` | JSON configurations and manifests used for core, extension, sensitivity, and ablation experiments |
| `results/` | Consolidated result tables, exported paper figures, validator-audit outputs, event-driven finality summaries, and audit-case artifacts |
| `paper/` | Manuscript source, compiled PDF, supplementary tables S1-S4, and BibTeX references |
| `docs/` | Reproducibility release plan and release manifest |

## Environment

Create and activate a Python environment, then install dependencies:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

For GPU execution, install a CUDA-enabled PyTorch build that matches the local CUDA driver before running the experiments. CPU execution is supported but slower.

## Reproducing Experiments

The full experiment suite is computationally expensive. The key entry points for generating and running the main benchmark are:

```powershell
python experiments/generate_b_journal_suite.py --suite core --profile cuda --out-dir configs\b_journal_suite_core_cuda
python experiments/run_b_journal_suite.py --manifest configs\b_journal_suite_core_cuda\core_manifest.txt
python experiments/summarize_b_journal_results.py --results-dir experiments_b_journal --out experiments_b_journal\summary_table.csv
```

For a smaller check, run a limited subset:

```powershell
python experiments/run_b_journal_suite.py --manifest configs\b_journal_suite_core_cuda\core_manifest.txt --limit 6
```

The released `results/` directory contains the consolidated summaries, paper tables, and figures used by the manuscript. The full raw run directories are not redistributed in this compact package; they can be regenerated by running the manifests above.

Note: `experiments_b_journal/` is a legacy local run-directory name retained by the experiment scripts for backward compatibility. It does not refer to the current target venue or manuscript scope.

## Regenerating Tables and Figures

To quickly verify the released summaries without re-running training, regenerate the paper tables and figures from the included CSV files:

```powershell
python experiments/export_paper_tables.py --summary results\summary_table.csv --out-dir verify_paper_tables
python experiments/export_paper_figures.py --tables-dir verify_paper_tables --out-dir verify_paper_figures
```

These scripts read the consolidated experiment outputs and regenerate the main paper tables and plots. Some scripts retain development defaults pointing to `experiments_b_journal/`; explicit paths are recommended for this release package.

The released validator-audit and protocol-cost outputs are included under `results/paper_tables/`, including:

- `overhead_table.csv`
- `validator_audit_scalability.csv`
- `validator_audit_network_latency.csv`
- `validator_audit_network_latency_summary.csv`
- `validator_audit_network_events.csv`
- `validator_audit_network_events_summary.csv`

The event-driven finality table in the manuscript is based on `validator_audit_network_events_summary.csv`; the corresponding simulation parameters are documented in Supplementary Table S4.

To regenerate overhead and validator-audit tables from scratch, first reproduce the raw experiment run directories with `config.json`, `summary.json`, and `audit_chain.jsonl`, then run:

```powershell
python experiments/export_overhead_table.py --summary experiments_b_journal\summary_table.csv --out experiments_b_journal\paper_tables\overhead_table.csv
python experiments/export_validator_audit_metrics.py --results-dir experiments_b_journal --out-dir experiments_b_journal\paper_tables
```

If only the released summary tables are needed, use the quick verification commands above instead of the raw-run export commands.

## Compiling the Manuscript

The released manuscript source is configured to use the figures and audit-case plots under `results/`:

```powershell
cd paper
pdflatex -interaction=nonstopmode -halt-on-error manuscript_neurocomputing.tex
bibtex manuscript_neurocomputing
pdflatex -interaction=nonstopmode -halt-on-error manuscript_neurocomputing.tex
pdflatex -interaction=nonstopmode -halt-on-error manuscript_neurocomputing.tex
```

The supplementary tables can be compiled separately from the same `paper/` directory:

```powershell
pdflatex -interaction=nonstopmode -halt-on-error supplementary_tables_neurocomputing.tex
pdflatex -interaction=nonstopmode -halt-on-error supplementary_tables_neurocomputing.tex
```

## Data

The experiments use public benchmark datasets: MNIST, Fashion-MNIST, and CIFAR-10. Datasets should be downloaded through the standard dataset loaders rather than redistributed in this repository.

## Notes

- The package is intended for paper reproducibility, not as a general-purpose FL framework.
- The validator layer is a threshold-finality simulation for evaluating verifiable aggregation records, not a full P2P deployment of PBFT, HotStuff, Tendermint, or a production blockchain network.
- Legacy exploratory result folders, local virtual environments, caches, and machine-specific paths are intentionally excluded.
- If this package is released publicly, tag the release version that corresponds to the submitted or accepted manuscript.
